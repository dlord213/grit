## Summary

Implemented an inline XP-style milestone tracker progress bar for the ExerciseDetailScreen history list. Each session card now displays a segmented horizontal gauge showing where that session's estimated 1RM falls relative to the user's all-time best. Sessions that set a new personal record trigger an animated "LEVEL UP!" badge with an elastic bounce-in effect.

---

## Feature Details

### Milestone Zone Logic
- 5 zones based on percentage of all-time best 1RM: Starter (0–50%), Novice (50–70%), Intermediate (70–85%), Advanced (85–95%), Elite (95–100%+)
- Each zone has a distinct color from the `GritTheme` palette
- Marker position = `clamp(sessionMax1RM / allTimeBest1RM, 0.0, 1.0)`

### Visual Components
- **Segmented bar**: 5 equal-width colored segments with rounded corners, separated by 2px gaps
- **Marker**: 14px circle with white border and zone-colored fill, positioned along the bar
- **Flag icon**: `Icons.flag_rounded` at the right end of the bar as the "goal"
- **Zone label**: Pill badge below the bar showing zone name (Starter/Novice/Intermediate/Advanced/Elite)
- **Session 1RM**: Text label showing the session's max estimated 1RM

### LEVEL UP Badge
- Animated gold-to-orange gradient pill badge with lightning bolt icon
- Uses `AnimatedScale` with `Curves.elasticOut` for a bounce-in effect on mount
- Only appears when a session's max 1RM exceeds all previous sessions' max 1RM

## Edge Cases Handled

- **Single session only**: Milestone bar hidden (nothing to compare against)
- **First session ever**: No LEVEL UP badge (no previous records)
- **Session with zero weight**: Position clamped to 0%, shows Starter zone
- **Session exceeds all-time best**: Position clamped to 100%, shows Elite zone
- **Multiple consecutive records**: Each new record after the first triggers LEVEL UP
- **Dark mode**: All zone colors are static constants that work on both light and dark backgrounds

## Key Code Patterns

### Pre-computing Session Data (chronological order)
```dart
// Reverse to chronological order for LEVEL UP detection
final List<String> chronologicalDates = dateKeys.reversed.toList();

// Compute per-date max 1RM
final Map<String, double> dateMax1RM = {};
for (final key in chronologicalDates) {
  double max1RM = 0;
  for (final s in groupedByDate[key]!) {
    final rm = _calculateSet1RM(s);
    if (rm > max1RM) max1RM = rm;
  }
  dateMax1RM[key] = max1RM;
}

// Detect LEVEL UP sessions
final Set<String> levelUpDates = {};
double bestSoFar = 0;
for (final key in chronologicalDates) {
  final rm = dateMax1RM[key]!;
  if (rm > bestSoFar && bestSoFar > 0) {
    levelUpDates.add(key);
  }
  if (rm > bestSoFar) bestSoFar = rm;
}
```

### Animated Badge Pattern
```dart
class _LevelUpBadgeState extends State<_LevelUpBadge> with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _scaleAnimation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 600),
      vsync: this,
    );
    _scaleAnimation = Tween<double>(begin: 0.0, end: 1.0).animate(
      CurvedAnimation(parent: _controller, curve: Curves.elasticOut),
    );
    _controller.forward();
  }
  // ... build uses Transform.scale with _scaleAnimation
}
```
