## Summary

Fixed dark mode issues across 5 screens where static `GritTheme.*` constants were used instead of `Theme.of(context)` equivalents, causing light colors to appear in dark mode. Also migrated all fonts from Nunito to Rubik.

---

## Issues Fixed

### 1. Dark Mode - Static Color References

**Root Cause**: Screens used `const TextStyle(color: GritTheme.textPrimary)` and `BoxDecoration(color: GritTheme.surface)` which are always light-theme values.

**Solution**: Replaced with `Theme.of(context)` equivalents:
- `GritTheme.background` â†’ `Theme.of(context).scaffoldBackgroundColor`
- `GritTheme.surface` â†’ `Theme.of(context).colorScheme.surface`
- `GritTheme.textPrimary` â†’ `Theme.of(context).colorScheme.onSurface`
- `GritTheme.divider` â†’ `Theme.of(context).dividerColor`

### 2. Font Migration

Changed all `fontFamily: 'Nunito'` references to `fontFamily: 'Rubik'` across theme.dart and all screen files. Updated `GoogleFonts.nunito()` calls to `GoogleFonts.rubik()`.

## Key Patterns Applied

### Dark Mode Pattern
```dart
// Before (broken in dark mode)
Container(
  decoration: BoxDecoration(
    color: GritTheme.surface,  // Always white
  ),
  child: Text('Hello', style: TextStyle(color: GritTheme.textPrimary)),  // Always dark purple
)

// After (works in both modes)
Container(
  decoration: BoxDecoration(
    color: Theme.of(context).colorScheme.surface,  // White in light, dark in dark
  ),
  child: Text('Hello', style: TextStyle(color: Theme.of(context).colorScheme.onSurface)),
)
```

### Font Pattern
```dart
// Before
GoogleFonts.nunito(color: color, fontSize: 16)
fontFamily: 'Nunito'

// After
GoogleFonts.rubik(color: color, fontSize: 16)
fontFamily: 'Rubik'
```
